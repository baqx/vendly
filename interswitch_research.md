# Interswitch API Integration Research

Interswitch is a leading payment gateway in Nigeria. For Vendly, we will integrate it to handle both card payments and potentially bank transfers (via their various products like Webpay or Quickteller).

## Key Components

### 1. Developer Credentials
- **ClientID:** Unique identifier for the application.
- **SecretKey:** Used for signing requests.
- **MerchantCode:** Assigned by Interswitch.
- **PayItemID:** Identifies the specific product/service being paid for.

### 2. Authentication Mechanism
Interswitch uses a custom security header for API requests.
- **Header:** `Authorization` (typically `InterswitchAuth [Base64(ClientID)]`)
- **Signature Headers:**
    - `Timestamp`: Current Unix timestamp.
    - `Nonce`: A unique random string for each request.
    - `Signature`: A SHA-512 hash of the combined request parameters.
    - `SignatureMethod`: Usually `SHA512`.

### 3. Signature Formula
The signature is typically generated by concatenating:
`HTTP_METHOD + URL + TIMESTAMP + NONCE + CLIENT_ID + SECRET_KEY + (Optional: request_body if POST)`
The resulting string is then hashed using SHA-512.

## Payment Flow

### Step 1: Initiate Payment
1. The backend (FastAPI) calculates the total and generates a unique `TransactionReference`.
2. The bot sends a payment link to the customer.
3. The customer is redirected to Interswitch's secure payment page (Webpay).

### Step 2: Transaction Verification
1. After payment, the customer is redirected back to a redirect URL.
2. The backend receives a `TransactionReference`.
3. The backend makes a server-to-server call to Interswitch's `status` endpoint to verify the payment.
    - **Endpoint:** `https://qa.interswitchng.com/api/v2/purchases?terminalid={terminalId}&amount={amount}&transactionreference={ref}` (Sandbox)
    - **Verification:** Check the `ResponseCode` (usually `00` for success).

### Step 3: Webhooks (Optional but Recommended)
- Interswitch can send asynchronous notifications to a predefined webhook URL when a payment is completed.
- This ensures the bot can update the order status even if the customer's browser is closed before the redirect happens.

## Implementation Details

### Backend (Python/FastAPI)
- Use `httpx` for making asynchronous API calls.
- Implement a utility function to generate the Interswitch security headers.
- Store sensitive credentials (`SecretKey`, `ClientID`) in environment variables.

### Frontend (Next.js)
- Handle the redirect response and display a "Success" or "Failure" message to the vendor/customer.
- Provide a dashboard for vendors to view transaction history via Interswitch API.
